# 异常处理

| 错误码 / 现象 | 展示给用户 | 处理方式 |
|---|---|---|
| 登录失效 / `status: error` / `AUTH_REQUIRED` / `AUTH_FAILED` | Token 不存在或已失效，请重新触发登录 | 执行 `login` 命令重新授权 |
| `10000` Token 服务端过期 | 登录已过期，需要重新授权 | 本地缓存可能仍存在但服务端已失效，需强制重新授权（`login --force`） |
| `10001` 城市未开通 | 该城市暂未开通服务，建议使用美团 App 确认是否支持 | — |
| `10002` 未知错误 | 信息异常，请检查下单参数。帮忙类用 businessType=1，帮买用 2 | 检查参数后重试 |
| `10112` 区域未开通 | 该地址所在区域暂未开通服务，建议更换到其他地址 | 更换地址 |
| `10113` 城市未开通 | 该地址所在城市暂未开通服务，建议切换到其他城市 | 更换城市 |
| `10114` 因恶劣天气/道路管制等暂无法配送 | 因恶劣天气/道路管制/疫情等原因，暂无法配送，敬请谅解，建议稍后重试 | 稍后重试 |
| `10115` 配送距离过远 | 配送距离过远，请重新选择地址 | 缩短配送距离 |
| `10116` 无效地址 | 请选择有效的地址，提供更精确的地址（含街道/楼号/校区名）有助于定位 | 提供更精确地址 |
| `10309` 配送费已动态发生变化 | 配送费已变动，请重新下单（同一次对话内重新预览即可） | 重新预览 |
| `10311` orderToken 失效 | 订单token失效，请重新下单（同进程内完成） | 使用 `preview_and_submit --confirm` 一体化命令 |
| `10401` 配送费计算失败 | 配送费计算失败，请重试一次 | 重试 |
| `11004~11008` 风控拒绝 | 下单信息异常，请检查地址和备注信息后重试 | 确保 `houseNumber`/`name`/`remark` 留空 |
| POI 搜索无结果 | 未找到该地址，请提供更精确的地址（含街道/楼号） | 换更精确关键词重新搜索 |
| 费用预览失败 | 费用预览失败，请检查地址坐标是否正确或尝试缩短距离 | 检查坐标或缩短距离 |
| 网络超时 | 网络请求超时，请检查网络后重试 | 检查网络/VPN 后重试 |
| `响应解析失败` | 接口响应异常，请检查网络/VPN 连接后重试 | 检查网络/VPN 连接 |

## Token 失效自动检测

当任何接口返回 `code: 10000` 且 `message` 为"未知错误"时，应视为 Token 服务端过期：

1. 自动执行 `login --force` 强制重新授权
2. 授权成功后重试原请求
3. 若重试仍失败，按实际错误码处理

> ⚠️ `login`（不带 `--force`）仅检查本地缓存文件，不验证服务端有效性。因此本地 Token 存在 ≠ 服务端有效，收到 10000 时必须使用 `login --force` 强制刷新。
